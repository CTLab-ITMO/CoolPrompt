# SGR-Niche Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make PE2+SGR genuinely win on its niche (verifiable-constraint flagship, reasoning secondary) via auto-enabled per-constraint feedback, a regression guard so it stops losing on easy tasks, and a portfolio router — without changing behavior for scalar-metric / single-method runs.

**Architecture:** Three phases. Phase 1 adds a `failure_breakdown()` capability to verifiable metrics (IFEval) and threads its output into the SGR proposer's diagnosis, auto-detected by duck-typing. Phase 2 makes the SGR loop respect "no edit needed" and prefer shorter prompts on ties. Phase 3 adds a `portfolio` method that runs PE2 and PE2+SGR and keeps the val-best. Each phase is evaluated against the committed 8-seed gpt-4o-mini PE2 baseline before continuing.

**Tech Stack:** Python 3.12, `uv`, LangChain, Pydantic, `unittest`, `flake8`. Spec: `docs/superpowers/specs/2026-05-25-sgr-niche-design.md`.

**Conventions (verified):** run via `uv run`; tests `uv run python -m unittest <module> -v`; lint `uv run --with flake8 flake8 <files>` (lines ≤ 79). Tests under `test/coolprompt/...`, invoked by exact module path (NOT `discover` — the `test/coolprompt` dir shadows the package). All edits must be no-ops for scalar metrics and PE2.

```python
    def failure_breakdown(self, outputs, targets):
        """Per-instruction failure summary for the diagnosis.

        Returns a string like
        'change_case:english_lowercase failed 3/5; ...' listing
        only the constraints that failed at least once, or None
        if every constraint passed / targets are unparseable.
        """
        import json
        fails = {}
        totals = {}
        for output, spec_json in zip(
            map(str, outputs), map(str, targets)
        ):
            try:
                spec = json.loads(spec_json)
            except (ValueError, TypeError):
                continue
            ids = spec.get("instruction_id_list", [])
            kw_list = spec.get("kwargs", [{}] * len(ids))
            for iid, kw in zip(ids, kw_list):
                totals[iid] = totals.get(iid, 0) + 1
                if not check_instruction(iid, output, kw or {}):
                    fails[iid] = fails.get(iid, 0) + 1
        if not fails:
            return None
        parts = [
            f"{iid} failed {fails[iid]}/{totals[iid]}"
            for iid in sorted(fails, key=lambda k: -fails[k])
        ]
        return "; ".join(parts)
```

```python
"""Best-of selection across optimizer methods (portfolio)."""


def select_portfolio(results):
    """Pick the method whose prompt scored highest on val.

    Args:
        results: dict method -> (prompt, val_score). Insertion
            order breaks ties (first listed wins).

    Returns:
        (method, prompt, val_score) of the winner.
    """
    best = None
    for method, (prompt, score) in results.items():
        if best is None or score > best[2]:
            best = (method, prompt, score)
    return best
```
